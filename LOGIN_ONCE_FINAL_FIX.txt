FINAL FIX: LOGIN ONCE AT THE START - REUSE SESSION
==================================================

✅ PERMANENTLY FIXED!

ROOT CAUSE ANALYSIS:
====================

The issue was NOT just about @BeforeSuite/@AfterSuite annotations.
The REAL problem was:

Each test was calling loginUser() which did:
1. landingpage.goTo() → Navigates to login page
2. landingpage.loginpage() → Logs in again
3. Waits for dashboard

When running 4 tests in sequence, this meant:
- Test 1: goTo() → loginpage() → Wait
- Test 2: goTo() → loginpage() → Wait
- Test 3: goTo() → loginpage() → Wait
- Test 4: goTo() → loginpage() → Wait

The repeated navigation/login was causing:
- Browser redirects
- Session conflicts
- Authentication issues
- Browser crash/close


SOLUTION: LOGIN ONCE IN @BeforeSuite, REUSE SESSION
===================================================

New Execution Flow:

@BeforeSuite setUp()
├─ Initialize ConfigReader
├─ Create ChromeDriver
├─ Maximize window
└─ Call loginToApplication() ← LOGIN ONCE HERE!
   ├─ Navigate to login page
   ├─ Enter credentials
   └─ Wait for dashboard

@Test testLogin()
├─ Already logged in ✓
└─ Execute test

@Test testAddProductToCart()
├─ Already logged in ✓
├─ Reuse browser session ✓
└─ Execute test

@Test testVerifyProductInCart()
├─ Already logged in ✓
├─ Reuse browser session ✓
└─ Execute test

@Test testCheckoutFlow()
├─ Already logged in ✓
├─ Reuse browser session ✓
└─ Execute test

@AfterSuite tearDown()
└─ Close browser (after ALL tests)


KEY CHANGES:
============

1. BaseTest.java:
   - Added loginToApplication() method
   - Called in @BeforeSuite setUp()
   - Logs in ONCE before all tests start

2. LandingpageBase.java:
   - Removed loginUser() private method (NO LONGER NEEDED)
   - Removed all loginUser() calls from @Test methods
   - Tests now assume user is already logged in
   - Tests reuse the same browser session

private void loginToApplication() {
    Landingpage landingpage = new Landingpage(driver);
    landingpage.goTo();
    landingpage.loginpage(configReader.getUserEmail(), 
                         configReader.getUserPassword());
    
    if (!landingpage.verifyLoginSuccess()) {
        String error = landingpage.getLoginError();
        throw new RuntimeException("Login failed: " + error);
    }
    System.out.println("✓ User logged in successfully!");
}

WHY THIS WORKS:
===============

1. Static WebDriver:
   - protected static WebDriver driver;
   - Shared across all tests
   - Single instance for entire suite

2. @BeforeSuite:
   - Runs ONCE before all tests
   - Sets up browser + login
   - Executed in main thread

3. @Test methods:
   - Run sequentially (parallel="false")
   - Reuse same browser instance
   - Reuse same login session

4. @AfterSuite:
   - Runs ONCE after all tests
   - Closes driver cleanly
